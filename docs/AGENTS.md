# AGENTS.md

Operating guide for agent work in this repo.

## Rules

- keep changes small and coherent
- do not revert user changes
- prefer apply_patch for file edits
- keep docs updated alongside structural changes
- follow the documented workflow and phase order

## Priority Docs

1. `DESIGN.md`
2. `canonical-workflow.md`
3. `initial-implementation-runthrough.md`
4. `v1-implementation-order.md`

## Non-Goals

- ad hoc architecture changes
- hidden setup steps
- deleting work that is already part of the plan
