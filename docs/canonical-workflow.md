# Canonical Workflow

Single source of truth for day-to-day development, git flow, quality gates, CI, and phase checks.

## Order

1. update from `main`
2. create a short-lived branch
3. implement one coherent task
4. run the right local checks
5. review the diff
6. stage intended files only
7. commit
8. run pre-push checks
9. push
10. let CI repeat the checks
