# v1 Implementation Order

Recommended order for the first production-ready version.

## Phases

- Phase 1: repo foundation
- Phase 2: visual system and runtime theme mapping
- Phase 3: web shell and product surface
- Phase 4: shared UI
- Phase 5: core app foundation (`config`, `db`, `auth`, health routes)
- Phase 6: admin app
- Phase 7: feature boundaries and route-to-feature composition
- Phase 8: DX baseline
- Phase 9: production hardening (standalone build, security headers, Docker runtime)
- Phase 10: reuse readiness
